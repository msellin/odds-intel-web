WC-D3-D4 — smoke description
============================

Scope: extend the WC win-probability curve with goal annotations (D3) and a
"next 10 minutes goal probability" widget (D4). Both surface during live WC
matches; pre-KO they show a kickoff placeholder; post-match the widget hides.

This file describes the smoke surface — it is read by humans/CI reviewing the
commit. The actual smoke harness in the engine repo
(`scripts/smoke_test.py`) is OUT OF SCOPE for this task per the constraints.

2. CHART BACKWARDS COMPATIBILITY
--------------------------------
Source: src/components/win-probability-chart.tsx

The chart accepts a new OPTIONAL `goals?: WpChartGoal[]` prop. When the prop
is omitted (every non-WC club call site) or passed as an empty array
(pre-tournament WC, or any match with no goals yet), the SVG renders exactly
as before — no diamonds, no extra DOM, no console noise.

The two existing call sites in src/app/(app)/matches/[id]/page.tsx:
  - Club path (line ~404): no `goals` prop → unchanged behaviour.
  - WC path (line ~666): passes `wcGoalAnnotations` (computed from
    matchEvents). For pre-tournament fixtures matchEvents is `[]`, so
    wcGoalAnnotations is `[]` and the chart still renders cleanly.


3. NEXT-10-MIN WIDGET
---------------------
Source: src/components/wc-next-10-min-goal.tsx
Math helper: src/lib/next-ten-goal.ts
  - exports nextTenGoalProb({ minute, homeXg, awayXg }): number | null
  - exports nextTenGoalVerdict(p): "Likely" | "Even" | "Unlikely"

  Formula: λ_per_min = (home_xg + away_xg) / max(minute, 1)
           λ_10      = λ_per_min * 10
           P(>=1)    = 1 - exp(-λ_10)

  Returns null when:
    - minute is null/undefined/≤0
    - both home_xg and away_xg are null
    - the accumulated total xG is 0
    - any input is non-finite

Render states:
  - status === "scheduled": placeholder card "Goal probability tracking starts at kickoff."
  - status === "live" + no live_xg snapshot: same placeholder
  - status === "live" + snapshot exists: bar (0-100%) + verdict line
  - status === "finished": component returns null (parent additionally guards)
